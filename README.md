# HIDShift

Rust firmware for switching USB HID keyboards, mice, and consumer-control
devices between Bluetooth Low Energy hosts through an ESP32-S3.

## Hardware

- ESP32-S3
- USB OTG D- on GPIO19
- USB OTG D+ on GPIO20
- Active-low control button on GPIO0
- A stable 5 V supply sized for the attached USB devices and hub

The firmware supports directly connected USB HID devices, composite HID devices,
and a single level of USB hubs.

## Build

Install the esp-rs Xtensa Rust toolchain by following the
[Rust on ESP toolchain instructions](https://docs.espressif.com/projects/rust/book/getting-started/toolchain.html).
Then load the environment file generated by `espup` for your shell. For a
default Unix installation:

```sh
source "$HOME/export-esp.sh"

cargo +esp build \
  -Zbuild-std=core,alloc \
  --release \
  --bin firmware \
  --features usb-host,ble-hid,storage \
  --target xtensa-esp32s3-none-elf
```

Do not enable the `diagnostic-input` feature in production builds.

## Flash

Flash the firmware with the included partition table:

```sh
espflash flash \
  --chip esp32s3 \
  --partition-table partitions/bridge.csv \
  --target-app-partition factory \
  target/xtensa-esp32s3-none-elf/release/firmware
```

## Controls

Actions are applied when the GPIO0 button is released.

| Button hold | Action |
| --- | --- |
| Less than 3 seconds | Select the next connected host |
| 3 to 8 seconds | Start pairing in the next available host slot |
| 8 seconds or longer | Remove the active host bond |

The pairing window remains open for 60 seconds or until pairing succeeds.

## Logs

Production builds emit logs at `info` level and above. Set `ESP_LOG` while
building to change the compile-time maximum log level:

```sh
ESP_LOG=debug cargo +esp build \
  -Zbuild-std=core,alloc \
  --release \
  --bin firmware \
  --features usb-host,ble-hid,storage \
  --target xtensa-esp32s3-none-elf
```

## Development Checks

Run the host-side checks before submitting changes:

```sh
cargo test
cargo fmt --check
cargo check --lib
```
